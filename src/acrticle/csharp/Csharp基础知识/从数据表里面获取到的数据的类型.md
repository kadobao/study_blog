## 使用强类型的Model类接收数据（推荐）

实际项目中更推荐使用强类型的Model类来接收数据，代码更清晰、易于维护。

**1. 定义Model类**

```csharp
public class ClockInRecord
{
    public string UserName { get; set; }
    public string Device { get; set; }
    public DateTime LastClockInTime { get; set; }
    public string RealName { get; set; }
    public string Shift { get; set; }
    public string OrderNo { get; set; }
}
```

**2. 使用强类型列表接收数据**

```csharp
List<ClockInRecord> records = new List<ClockInRecord>();
```

**3. 访问数据**

```csharp
foreach (var record in records)
{
    string name = record.RealName;  // 直接访问，无需类型转换
    DateTime time = record.LastClockInTime;
}
```

**强类型Model的优点：**
- 有编译时类型检查，减少运行时错误
- 智能提示更友好，开发效率高
- 便于后续维护和重构
