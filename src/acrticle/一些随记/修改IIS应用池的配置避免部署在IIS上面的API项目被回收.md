# IIS应用池配置优化

## 概述

在生产环境中，IIS应用池的默认配置可能会导致API项目被意外回收，影响服务的稳定性。通过调整应用池的相关配置参数，可以有效避免此类问题。

## 配置步骤

### 1. 打开应用池高级设置

右键点击需要修改的应用池，选择 `高级设置` 选项。

### 2. 配置回收参数

在弹出的高级设置窗口中，进行以下关键配置：

| 配置项 | 位置 | 设置值 | 说明 |
|--------|------|--------|------|
| 固定时间间隔 | 回收 - 固定时间间隔 | 0 | 禁用定期回收，避免固定时间间隔导致的应用重启 |
| 闲置超时（分钟） | 进程模型 - 闲置超时（分钟） | 0 | 禁用闲置超时，防止应用因空闲被回收 |

#### 设置固定时间间隔

将 `回收 - 固定时间间隔` 的值设置为 **0**

![避免IIS回收_1](/assets/images/避免IIS回收_1.png)

#### 设置闲置超时

将 `进程模型 - 闲置超时（分钟）` 的值设置为 **0**

![避免IIS回收_2](/assets/images/避免IIS回收_2.png)

## 注意事项

> **重要提醒**：修改应用池配置需要重启相关应用池才能生效。
