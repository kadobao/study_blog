## 问题背景

IIS 部署项目时，默认会开启日志功能，记录着每一次请求的详细信息，包括：

- 请求时间
- 请求方法
- 请求路径
- 请求参数
- 请求头
- 请求体
- 响应状态码
- 响应头
- 响应体

## 日志存储位置

这些日志文件默认存储在 `C:\inetpub\logs\LogFiles` 目录下，每个网站对应一个日志目录，目录名就是网站的域名或 IP 地址。

## 日志文件大小限制

每个日志文件的大小默认是 `100 MB`，当文件大小超过限制时，会自动创建一个新的日志文件。

## 问题影响

当项目增多，或者访问量增加时，日志文件会不断增长，导致 C 盘空间被占满。

## 解决方案

关闭 IIS 日志可以有效防止磁盘空间被快速占用。

### 关闭日志的步骤

1. 在 IIS 管理器中，点击左侧的服务器节点（即计算机名，代表整个服务器级别）
2. 在中间的功能视图中，双击打开"日志"
3. 在右侧操作面板中，点击"关闭日志"
4. 确认关闭日志
